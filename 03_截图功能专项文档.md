# ZBB 截图功能专项文档

## 一、需求背景

在企业微信自动化流程中，需要在特定步骤（如报备失败时）截图保存当前界面，用于问题排查和记录。

---

## 二、尝试的方案

### 方案1：AccessibilityService 的 takeScreenshot

**实现方式**：
```kotlin
// AccessibilityServiceImpl.kt
fun takeScreenshot(): String? {
    val rootNode = service?.rootInActiveWindow ?: return null
    val screenshot = Bitmap.createScreenshot(
        rootNode, 
        AccessibilityNodeInfo.FLAG_PREFETCH_DISABLED
    )
    // 保存到应用私有目录
    val file = File(cacheDir, "screenshot_${System.currentTimeMillis()}.png")
    FileOutputStream(file).use { screenshot.compress(Bitmap.CompressFormat.PNG, 100, it) }
    return file.absolutePath
}
```

**问题**：
- 在 WebView/小程序界面中截图返回 null
- 可能被应用的安全策略限制

**结果**：❌ 不可用

---

### 方案2：React Native 的截图 API

**尝试使用**：
- `expo-camera` 的截图功能
- 屏幕录制 API

**问题**：
- 这些 API 主要用于相机/录屏，不适合截取当前屏幕
- 在小程序界面中同样失效

**结果**：❌ 不适用

---

### 方案3：三指下滑手势

**实现方式**：
```kotlin
// 模拟三指下滑（系统级截图快捷键）
val gesture = GestureDescription.Builder()
for (i in 0..2) {
    val path = Path().apply { moveTo(x, y); lineTo(x, y - 500) }
    gesture.addStroke(GestureDescription.StrokeDescription(path, 0, 300))
}
dispatchGesture(gesture.build(), callback)
```

**问题**：
- 在某些 Android 版本/设备上需要额外权限
- 小程序 WebView 可能拦截手势

**结果**：❌ 不可靠

### 方案4：screencap Shell 命令（最终方案）

**实现方式**：
```kotlin
// AccessibilityServiceImpl.kt
fun screencapShell(filePath: String): String? {
    try {
        // 确保目录存在
        Runtime.getRuntime().exec("mkdir -p ${File(filePath).parent}")
        
        // 执行截图命令
        val cmd = "screencap $filePath"
        val process = Runtime.getRuntime().exec(arrayOf("sh", "-c", cmd))
        val exitCode = process.waitFor()
        
        if (exitCode == 0) {
            return filePath
        }
        return null
    } catch (e: Exception) {
        Log.e("Automation", "截图失败: ${e.message}")
        return null
    }
}
```

**暴露给 React Native**：
```kotlin
// AutomationModule.kt
@ReactMethod
fun screencapShell(filePath: String, promise: Promise) {
    Thread {
        val result = service?.screencapShell(filePath)
        promise.resolve(result)
    }.start()
}
```

**TypeScript 封装**：
```typescript
// native/index.ts
screencapShell: (filePath: string): Promise<string | null> => {
    return new Promise((resolve) => {
        NativeModules.ZbbAutomation.screencapShell(filePath, (result: string | null) => {
            resolve(result);
        });
    });
}
```

## 三、遇到的问题及解决方案

### 问题1：路径权限

**问题**：`/storage/emulated/0/` 在 APK 中不可访问

**解决方案**：改用 `/sdcard/`
```kotlin
// 原来
val filePath = "/storage/emulated/0/Pictures/ZBB/screenshot.png"

// 修改后
val filePath = "/sdcard/Pictures/ZBB/screenshot.png"
```

**结果**：✅ ADB 测试成功

---

### 问题2：目录不存在

**问题**：截图时目录不存在导致失败

**解决方案**：添加 `mkdir -p` 命令
```kotlin
Runtime.getRuntime().exec("mkdir -p ${File(filePath).parent}")
```

**结果**：✅ 目录自动创建

---

### 问题3：APK 中执行失败但 ADB 成功

**现象**：
- ADB 命令 `adb shell screencap /sdcard/Pictures/ZBB/test.png` 成功
- APK 中执行失败，返回 null

**可能原因**：
1. APK 中的 Runtime.exec 与 ADB shell 环境不同
2. 权限问题（需要 WRITE_EXTERNAL_STORAGE 或媒体存储权限）

**状态**：⚠️ 待解决

### 方案A：使用 MediaStore API
```kotlin
fun screencapMediaStore(fileName: String): Uri? {
    val contentValues = ContentValues().apply {
        put(MediaStore.Images.Media.DISPLAY_NAME, fileName)
        put(MediaStore.Images.Media.MIME_TYPE, "image/png")
        put(MediaStore.Images.Media.RELATIVE_PATH, "Pictures/ZBB")
    }
    return contentResolver.insert(MediaStore.Images.Media.EXTERNAL_CONTENT_URI, contentValues)
}
```
